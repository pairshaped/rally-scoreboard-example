# Scoreboard Unified

Scoreboard Unified is the single-source projection spike for the scoreboard app.

It is working toward one authored Gleam source tree that can generate the split client and server packages used by the sibling scoreboard app. The domain stays intentionally small: public scores, standings, team pages, sign in, and an admin score desk.

This project is not the finished projector. Today it has two useful pieces:

- `.generated/` contains the current split-package target shape copied from the sibling app.
- root `src/` is the new authored source area. Proute already generates route and page glue from it.

## Shape

- `src/public/pages/**` and `src/admin/pages/**` are the root page trees.
- `src/generated/proute/{public,admin}/` is generated by Proute.
- `.generated/client` and `.generated/server` are the current generated-target packages.

The root page modules are thin scaffolds for now. Public scoreboard pages and the admin score desk already render through the real target-neutral shared views; sign-in and not-found are still simple placeholders. That gives the future projector real app-shaped source without pretending the hard client/server split is done.

The next source generators should run before projection:

- Marmot writes typed SQL under `src/generated/sql/...`.
- Proute writes route and page glue under `src/generated/proute/...`.
- The future Rust projector copies or rewrites reachable root modules into `.generated/{client,server}`.

From the repository root:

- `gleam run -m marmot migrate`
- `gleam run -m marmot`
- `gleam run -m proute`
- `gleam check`

The generated chase target should also stay runnable:

- `(cd .generated/client && gleam test)`
- `(cd .generated/server && gleam test)`

Admin routes are generated from `src/admin/pages`.

## Projection Target

The generated target still needs to preserve the sibling app's core choices:

- root `ToServer` and `ToClient` wire contracts
- compact ETF payloads
- true client hydration
- page-local client/server models where useful
- no page-local wire/domain types that force ETF namespace work

The target is not just a shape exercise. `.generated/` should be a functioning scoreboard app after generation. The current copy passes client and server test suites; the future Rust projector should keep that true while replacing the copied files with generated output.
